# Bandit Baseline Governance

## Overview

The `.bandit.yaml` file is the security baseline for Phase 5a (PR A). It documents known acceptable security findings (or findings deferred for future hardening phases).

## Review Cadence

This baseline should be reviewed and refreshed:
- At the start of every new phase plan
- Anytime a flagged pattern is either fixed (remove from allow-list) or newly introduced during a phase

## Refresh Command

```bash
bandit -r backend/ --exclude backend/scripts,backend/migrations -ll -iii -f yaml -o .bandit.yaml
```

## Flags Explained

- `-ll`: Show MEDIUM and above severity (Bandit's `-l` flag is cumulative "show LOW+"; `-ll` means "show MEDIUM+")
- `-iii`: Requires HIGH confidence
- **Effective filter**: MEDIUM+ severity AND HIGH confidence only
