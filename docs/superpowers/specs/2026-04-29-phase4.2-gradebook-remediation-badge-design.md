# Phase 4.2 #7 — Gradebook Remediation Badge

**Status:** Green-tier item from `project_phase4.2_backlog.md` #7.
**Differentiator vs Progress Learning:** PL doesn't visually distinguish remediations from regular assessments in its gradebook view. Graider's gradebook now surfaces a `Remediation` (and `Recalled` when applicable) pill under each column header, giving teachers an at-a-glance audit signal as they scan grades.

## Goal

Teacher scans the Gradebook, sees pills under column headers identifying which assessments are remediations and which of those are recalled. Same pills appear in the SubmissionDetail drawer header for consistency. Read-only signal; no filter, no grouping, no sort change.

## Locked decisions (from brainstorming, 2026-04-29)

- **Q1 → A.** Badge ONLY. No filter dropdown, no grouping change. Smallest meaningful slice; teachers can mentally filter by reading the badges. A 2-D student × assessment grid doesn't lend itself cleanly to column-hide filters anyway.
- **Q2 → A.** Pill BELOW the column header title. Preserves horizontal width on a dense grid (column `<th>` already has `minWidth: 100px`). NOT inline beside the title; NOT replacing the title with a "Remediation:" prefix.
- **Q3 → B.** BOTH pills when applicable. A recalled remediation column shows `Remediation` + `Recalled`, not one or the other. Future-safe (doesn't conflate "recalled" with "remediation"); explicit audit signal that this column WAS a remediation that was recalled.
- **Q4 → A.** SubmissionDetail drawer header also shows the badge(s). Same pills, rendered below `content_title` (not next to `student_name` — the badges describe the content, not the student).

### Frontend

**New file: `frontend/src/components/RemediationBadges.jsx`** (~30 LOC):

Single source of truth for the two pills. Used by Gradebook column headers AND SubmissionDetail drawer header.

```jsx
import React from "react";

function isRemediation(item) {
  // Defensive predicate: render the badge only when target_student_ids is a
  // non-empty array. Backend dashboard's spec contract is "non-null"; on the
  // frontend we tighten to "non-empty array" because publish_to_class rejects
  // empty arrays at write time, AND because legacy/cached responses with
  // undefined fields would otherwise render a phantom badge.
  return Array.isArray(item && item.target_student_ids) &&
         item.target_student_ids.length > 0;
}

function isRecalled(item) {
  return isRemediation(item) && item.is_active === false;
}

export default function RemediationBadges({ item }) {
  if (!isRemediation(item)) return null;
  return (
    <div style={{ display: "flex", flexWrap: "wrap", gap: "4px", justifyContent: "center" }}>
      <span style={{
        fontSize: "0.78rem", padding: "3px 10px", borderRadius: "6px",
        background: "rgba(99,102,241,0.15)", color: "var(--accent-primary)", fontWeight: 700,
      }}>Remediation</span>
      {isRecalled(item) && (
        <span style={{
          fontSize: "0.78rem", padding: "3px 10px", borderRadius: "6px",
          background: "rgba(239,68,68,0.12)", color: "var(--danger)", fontWeight: 700,
        }}>Recalled</span>
      )}
    </div>
  );
}
```

Style values match the existing dashboard recall badge in `RemediationEffectiveness.jsx:188` exactly (Phase 4.2 #5 precedent). Color split: `var(--accent-primary)` (indigo tint) for Remediation; `var(--danger)` (red tint) for Recalled.

**`frontend/src/tabs/Gradebook.jsx`** (modified, ~+10 LOC):
- Import `RemediationBadges`.
- Replace the column header at line ~117 from `<th>{a.title}</th>` to a vertical-flex stack:

  ```jsx
  <th key={a.content_id} style={{ /* existing styling */ }}>
    <div style={{ display: "flex", flexDirection: "column", alignItems: "center", gap: "4px" }}>
      <span>{a.title}</span>
      <RemediationBadges item={a} />
    </div>

  ```

The `<RemediationBadges>` component returns `null` when the item isn't a remediation, so non-remediation columns still render with just the title (no extra vertical space, no empty-pill placeholder).

**`frontend/src/tabs/SubmissionDetail.jsx`** (modified, ~+5 LOC):
- Import `RemediationBadges`.
- In the drawer header at line ~77 (where `content_title` renders), add `<RemediationBadges item={data} />` immediately below the content title. The drawer's response root (`data`) now has `target_student_ids` and `is_active` at top level (per the backend extension above), so the same component works.
- Place below `content_title`, NOT next to `student_name` — the badges describe the content, not the student.

### Edge cases

| Case | Behavior |
|---|---|
| Column with `target_student_ids === null` (regular class-wide assessment) | `<RemediationBadges>` returns null → column renders just the title. No vertical-space penalty. |
| Column with `target_student_ids === []` (shouldn't exist; `publish_to_class` rejects this) | Defensive `length > 0` check → no badge. |
| Column with `target_student_ids === undefined` (legacy/cached response missing the field) | `Array.isArray(undefined) → false` → no badge. Backwards-compatible without explicit migration. |
| Column with valid remediation + `is_active === true` | Single `Remediation` pill. |
| Column with valid remediation + `is_active === false` | Two pills: `Remediation` + `Recalled`. |
| Column with `is_active === false` AND `target_student_ids === null` (a non-remediation assessment that was deactivated through some legacy path) | NO badge. We don't badge non-remediation inactives — no current UI surfaces "assessment inactive" elsewhere; conflating that here would be inconsistent. |
| Recall happens AFTER the gradebook page loaded | Badge is stale until the next fetch. Current Gradebook doesn't poll; teacher refreshing the analytics tab triggers re-fetch. Acceptable for green-tier scope; not worth adding cache invalidation infrastructure. |
| SubmissionDetail drawer opens for a recalled-remediation submission | Drawer header shows both pills. Submission data still loads normally — recall hides from students, not from teachers. |
| Long column titles wrapping with badges below | Existing `<th>` styling already handles title wrapping; the vertical flex container just adds the badges below. No layout regression. |
| Mobile / narrow viewport | Existing horizontal `overflow-x: auto` on the gradebook table preserves scrolling; badges add ~26px of vertical height per column header. Acceptable. |
| Backend response missing `is_active` or `target_student_ids` (e.g., older deployment) | Defensive predicate handles undefined gracefully. No explicit error or warning surfaced. |
