## Goal

Make `standards_mastery` (the JSONB rollup at `student_submissions.results.standards_mastery`) carry per-DOK breakdown in addition to the existing aggregate so consumers can answer the load-bearing pedagogical question: "did the DOK 3 remediation actually move DOK 3 mastery, or only DOK 1 recall?"

This is the structural piece that makes DOK *meaningful* rather than decorative. Sprint 1 (PR #151) made DOK *visible*. Sprint 3 (#3 effectiveness DOK filter) and beyond use the data this sprint produces.

## New JSONB shape

### Old (current, in production)

```json
{
  "MA.6.AR.1.2": {
    "percentage": 80,
    "points_earned": 8,
    "points_possible": 10,
    "question_count": 2
  }
}
```

### New (Sprint 2)

```json
{
  "MA.6.AR.1.2": {
    "overall": {
      "percentage": 80,
      "points_earned": 8,
      "points_possible": 10,
      "question_count": 2
    },
    "by_dok": {
      "1": {"percentage": 100, "points_earned": 4, "points_possible": 4, "question_count": 1},
      "3": {"percentage": 67, "points_earned": 4, "points_possible": 6, "question_count": 1}
    }
  }
}
```

**Rules:**

1. `overall` is always present and equals what the old flat shape carried (sum of all questions, regardless of DOK presence).
2. `by_dok` only contains keys for DOKs that had at least one contributing question (1-4). DOK keys are JSON strings (`"1"`, `"3"`) but logically integers; readers normalize via `_validate_dok`.
3. Questions with missing or invalid DOK count toward `overall` but NOT toward any `by_dok` bucket. This preserves mastery totals while refusing bad categorization.
4. `by_dok` may be empty `{}` — legacy submissions, or new submissions where no question carried a DOK field.

## Architecture

### Write path

```
AI-graded question (q.dok set) ──┐
                                  │
Instant-graded question (q.dok)  ─┼──→ q.get('dok') in question_results
                                  │
                                  ▼
                  _build_standards_mastery(question_results)
                                  │
                                  ▼
                  results.standards_mastery (NEW shape)
```

Currently the per-question result writers in `portal_grading.py` (3 sites — Sprint 1) AND `grading_service.py` (2 sites — Codex MAJOR finding for Sprint 2) need to preserve `q.get('dok')`. Sprint 1 covered the 3 portal_grading sites; Sprint 2 adds the 2 grading_service sites.

### Read path

```
JSONB (old or new shape, possibly malformed)
              │
              ▼
   _normalize_mastery_shape(raw)        ◄── single boundary adapter
              │
              ▼
   _sanitize_standards_mastery(sub)     ◄── retrofitted to call adapter inline
              │
              ▼
   _aggregate_mastery_for_student(...)  ◄── helper now operates on normalized internals
              │
              ▼
   Route response                       ◄── routes choose: project down to flat, OR keep new shape
```

### API response shape policy (load-bearing decision — Codex DISPUTE/MINOR)

**Explicit policy:** Internals always normalize to new shape. Route responses preserve their existing shape unless they explicitly need `by_dok`.

| Route | Response shape | Frontend impact |
|-------|----------------|-----------------|
| Progress Rank | Flat (existing) | None |
| Student Report Card | **NEW** — includes `by_dok` per standard | Renders new section |
| Assessment Comparison | Flat (existing) | None |
| Effectiveness Dashboard | Flat (existing) | None (Sprint 3 changes this) |
| Submission Detail | Flat (existing) | None |
| post_remediate (red-tier check) | No mastery in response | None |
| Gradebook | No mastery in response | None |

This means:
- `_aggregate_mastery_for_student(..., include_dok=False)` (default) returns flat shape — preserves existing API contracts.
- `_aggregate_mastery_for_student(..., include_dok=True)` returns new shape with `by_dok`.
- Only Student Report Card route uses `include_dok=True`.

This avoids silently changing API responses for routes whose frontend isn't ready.

**`_build_standards_mastery(question_results)`** — replace flat shape construction with new shape:

```python
def _build_standards_mastery(question_results):
    """Roll up per-question scores into the per-standard mastery dict.

    Phase 4.3 Sprint 2: emits new shape `{standard: {overall, by_dok}}`.
    `overall` = aggregate across all questions (matches old flat shape values).
    `by_dok` = subset by valid DOK level; questions with missing/invalid dok
              count toward `overall` but NOT any `by_dok` bucket.
    """
    by_standard = {}
    for q in question_results or []:
        std = q.get('standard')
        if not std:
            continue
        agg = by_standard.setdefault(std, {
            'overall': _new_mastery_agg(),
            'by_dok': {},
        })
        _accumulate(agg['overall'], q)
        dok = _validate_dok(q.get('dok'))
        if dok is not None:
            d_agg = agg['by_dok'].setdefault(dok, _new_mastery_agg())
            _accumulate(d_agg, q)
    return _finalize_mastery(by_standard)
```

Helpers (private to grading_service):
- `_new_mastery_agg()` returns `{'points_earned': 0, 'points_possible': 0, 'question_count': 0}`.
- `_accumulate(agg, q)` adds `q.points_earned` / `q.points_possible` / `+1` to agg.
- `_finalize_mastery(by_standard)` walks the tree and adds `percentage = round(100 * earned / max(1, possible))` to every aggregator.

`_validate_dok` is reused from `student_portal_routes.py` (Sprint 1). To avoid a circular import, **move `_validate_dok` to a shared location.** Options:
- Move to `backend/services/grading_service.py` (alongside the builder that uses it)
- Move to a new `backend/services/dok.py` (cleanest separation)

**Default: move to `grading_service.py`** — it's the only consumer module-internal; if more places need it later, refactor then. `student_portal_routes.py` imports it from grading_service.

#### `backend/routes/student_portal_routes.py`

**`_normalize_mastery_shape(raw)`** — new boundary adapter (Codex EXTEND):

```python
def _normalize_mastery_shape(raw):
    """Convert old flat shape → new {overall, by_dok} shape; pass new shape through.

    Always returns a dict with `overall` (with at least the 4 known keys) and
    `by_dok` (possibly empty). Returns None if raw is malformed.
    """
    if not isinstance(raw, dict):
        return None
    if 'overall' in raw:
        # New shape — validate and return as-is (sanitize sub-structures).
        overall = raw.get('overall') if isinstance(raw.get('overall'), dict) else {}
        by_dok_raw = raw.get('by_dok') if isinstance(raw.get('by_dok'), dict) else {}
        by_dok = {}
        for k, v in by_dok_raw.items():
            normalized_k = _validate_dok(k)
            if normalized_k is not None and isinstance(v, dict):
                by_dok[normalized_k] = v
        return {'overall': overall, 'by_dok': by_dok}
    # Old flat shape — wrap.
    return {'overall': dict(raw), 'by_dok': {}}
```
