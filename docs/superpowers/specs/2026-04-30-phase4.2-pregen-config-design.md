# Phase 4.2 #3 — Pre-generation Config Dialog for Remediation

**Status:** Green-tier item from `project_phase4.2_backlog.md` #3 — the deferred Q4 → B from Phase 4.
**Differentiator vs Progress Learning:** PL has a fixed remediation question count and difficulty. Graider's drawer now opens to a pre-generation config step where the teacher picks count (3-15) and difficulty (easier / same / harder) before AI generation runs. One extra click; teacher control over rigor + length.

## Goal

When the teacher opens the remediation drawer (cell click or column-header), the drawer opens in a NEW `"config"` state showing a question-count slider (3-15, default 8) and a three-button difficulty toggle (easier / same / harder, default same). Clicking "Generate" sends `{count, difficulty}` to `/remediate`, which uses these params to parameterize the AI prompt. Both shared-mode and personalized-mode honor the params. Backwards-compat: callers without `count`/`difficulty` get current behavior (8 + same).

## Locked decisions (from brainstorming + Codex mid-brainstorming reconciliation, 2026-04-30)

- **Q1 (config UI location):** New `"config"` state in `RemediationDrawer.jsx`. Drawer opens to config (not auto-fetch); teacher picks → clicks Generate → goes to "generating" → "preview" → ...
- **Q2 (difficulty options):** Three buttons — easier / same / harder. NOT a 5-point slider.
- **Q3 (count granularity):** Slider 3-15 (continuous integer).
- **Q4 (defaults):** count=8, difficulty=same (preserves current behavior).
- **Q5 (mode coverage):** Both single-student AND class-wide go through config. Consistency.
- **Q6 (skip-config preference):** None. 1 click is tiny; localStorage state surface not justified.
- **Q9 (personalized mode interaction):** Config picked ONCE; ALL N variants in personalized mode honor the same count + difficulty. No per-variant config.
- **Q10 (cost warning at high count):** None. Trust teachers.
- **Q11 (API shape):** `postRemediate(classId, {standard_code, target_mode, target_student_id?, count, difficulty})`. Server-side defaults if missing.
- **Strict validation (Codex MAJOR):** explicit invalid values → 400; defaults only when missing. Reject booleans (Python `bool` is `int` subclass). Validate BEFORE red-tier resolution + cap check.
- **Both prompt paths parameterized (Codex MAJOR):** `_build_remediation_prompt(grade, subject, standard_code, count, difficulty)` is the single source of truth. The existing inline shared-mode `base_prompt` at line ~2709 is REMOVED in favor of calling the helper.
- **Response metadata (Codex MAJOR):** `/remediate` response echoes `count` and `difficulty` so the drawer can render "Generating N...", regenerate reuses params, and personalized variants are debug-traceable.
- **Grade arithmetic clamping (Codex MINOR):** numeric grade clamped to K-12; non-numeric grades use generic "above/below current grade level" phrasing.

## Architecture

### Backend

**File:** `backend/routes/student_portal_routes.py`.

#### Update `_build_remediation_prompt`

```python
DIFFICULTY_OPTIONS = ('easier', 'same', 'harder')
REMEDIATION_COUNT_MIN = 3
REMEDIATION_COUNT_MAX = 15
REMEDIATION_COUNT_DEFAULT = 8
REMEDIATION_DIFFICULTY_DEFAULT = 'same'


def _difficulty_directive(difficulty, grade):
    """Map (difficulty, grade) → prompt directive line. Clamps grade
    arithmetic to K-12 (Codex MINOR — no `grade-13` references).
    For non-numeric grades, uses generic phrasing."""
    grade_num = None
    try:
        grade_num = int(str(grade).strip())
    except (ValueError, TypeError, AttributeError):
        pass

    if difficulty == 'easier':
        if grade_num is not None:
            target = max(1, grade_num - 1)  # K-clamp at 1
            return f"Use simpler vocabulary and more scaffolding. Aim for grade-{target} difficulty."
        return "Use simpler vocabulary and more scaffolding. Aim for content below the current grade level."
    if difficulty == 'harder':
        if grade_num is not None:
            target = min(12, grade_num + 1)  # 12-clamp
            return f"Use more challenging vocabulary and higher cognitive demand. Include some grade-{target} concepts."
        return "Use more challenging vocabulary and higher cognitive demand. Include some content above the current grade level."
    # 'same' (default)
    return "Difficulty: grade-level review."


def _build_remediation_prompt(*, grade, subject, standard_code, count, difficulty):
    """Phase 4.2 #1 + #3: build the AI prompt for both shared and
    personalized modes. Single source of truth (Codex MAJOR — earlier
    version had a duplicate inline base_prompt in shared-mode path).
    """
    mc = round(count * 0.6)
    sa = count - mc
    difficulty_directive = _difficulty_directive(difficulty, grade)
    return (
        f"Generate exactly {count} grade-{grade} {subject} practice questions aligned to "
        f"standard {standard_code}, AND a short lesson explaining the standard.\n\n"
        f"QUESTIONS: Mix of {mc} multiple-choice questions (4 choices each, "
        f"exactly 1 correct, marked with an 'answer' field whose value is either the "
        f"choice letter (A/B/C/D) or the choice text) and {sa} short-answer questions "
        f"(each with an 'answer' field containing the model answer). "
        f"Each question MUST include a 'standard' field equal to '{standard_code}'. "
        f"{difficulty_directive}\n\n"
        f"LESSON: A 300-400 word total mini-lesson with three string fields: "
        f"`intro` (~80-120 words explaining what standard {standard_code} is and "
        f"why it matters), `worked_example` (~150-200 words: a single fully worked "
        f"problem with problem statement, step-by-step reasoning, and final answer), "
        f"and `key_takeaway` (1-2 sentence summary of the rule or principle). "
        f"The worked example MUST demonstrate the same concept the practice "
        f"questions test. For any math, use \\(...\\) for inline equations and "
        f"\\[...\\] for display equations (NOT dollar signs — the renderer does "
        f"not support them).\n\n"
        f"Return valid JSON of this exact shape: "
        f"{{\"title\": \"Practice - {standard_code}\", "
        f"\"lesson\": {{\"intro\": \"...\", \"worked_example\": \"...\", "
        f"\"key_takeaway\": \"...\"}}, "
        f"\"sections\": [{{\"name\": \"Practice\", \"questions\": [...]}}]}}"
    )
```

#### Update `/remediate` route validation

Add (BEFORE red-tier resolution and cap check):

```python
# Phase 4.2 #3: validate count + difficulty.
raw_count = body.get('count')
raw_difficulty = body.get('difficulty')

if raw_count is None:
    count = REMEDIATION_COUNT_DEFAULT
else:
    # Reject booleans explicitly (Python bool is an int subclass —
    # otherwise True passes int() and becomes count=1).
    if isinstance(raw_count, bool) or not isinstance(raw_count, int):
        return error_response(
            f"count must be an integer between {REMEDIATION_COUNT_MIN} and {REMEDIATION_COUNT_MAX}",
            400,
        )
    if raw_count < REMEDIATION_COUNT_MIN or raw_count > REMEDIATION_COUNT_MAX:
        return error_response(
            f"count must be between {REMEDIATION_COUNT_MIN} and {REMEDIATION_COUNT_MAX}",
            400,
        )
    count = raw_count

if raw_difficulty is None:
    difficulty = REMEDIATION_DIFFICULTY_DEFAULT
elif raw_difficulty in DIFFICULTY_OPTIONS:
    difficulty = raw_difficulty
else:
    return error_response(
        "difficulty must be one of: " + ", ".join(DIFFICULTY_OPTIONS),
        400,
    )
```

### Frontend

**File:** `frontend/src/tabs/RemediationDrawer.jsx`.

#### State machine extension

New state: `"config"`. State transitions:
- `idle` → `config` (on drawer open — REPLACES the current auto-`generating` transition)
- `config` → `generating` (on Generate click)
- `generating` → `preview` | `error` (existing)
- `preview` → `regenerating` (existing — uses CURRENT params; no config re-prompt)
- `preview` → `config` (NEW — when teacher clicks "Adjust settings"; lets them pick new count/difficulty without abandoning the drawer)

#### New state slots

```js
var [configCount, setConfigCount] = useState(8);
var [configDifficulty, setConfigDifficulty] = useState("same");
```

Reset to defaults on every drawer open (not sticky).

#### useEffect on `open`

Currently auto-fires `postRemediate`. Change to: reset state and set `state="config"`. The `postRemediate` call moves into a new `handleGenerate()` function called when the teacher clicks Generate.

#### Config UI

In `state === "config"`, render:
- Heading: "Configure remediation"
- Question count slider 3-15, displaying current value
- Three-button difficulty toggle (easier / same / harder), highlighted on selection
- Generate button (disabled when in-flight)
- Cancel button (closes drawer)

Match existing drawer styling (glass-card chrome, var(--accent-primary) for active states).

#### "Generating N..." copy

Replace hardcoded `"Generating 8 practice questions..."` with `"Generating " + configCount + " practice questions..."`. Uses the user's chosen count.

#### `postRemediate` payload

Extend to include `count` and `difficulty`:

```js
var payload = {
  standard_code: standardCode,
  target_mode: targetMode,
  count: configCount,
  difficulty: configDifficulty,
};
if (targetMode === "single_student") payload.target_student_id = targetStudentId;
```

Both `handleGenerate()` and `regenerateAll()` use this same payload.

#### "Adjust settings" button

In the preview state's footer (next to "Regenerate all"), add a third button: "Adjust settings". Clicking it returns to `state="config"`, preserving `data` and `variants` in case the teacher cancels back to preview.

### Edge cases

| Case | Behavior |
|---|---|
| Teacher opens drawer, never clicks Generate | Drawer renders config UI; closes via Cancel or X. No AI cost incurred. |
| Teacher slides count to 3 | Server generates exactly 3. Mix: 2 MC + 1 SA. 422 floor still requires ≥3 valid questions after post-processing. |
| Teacher slides count to 15 | Server generates exactly 15. Mix: 9 MC + 6 SA. Larger AI cost, no warning. |
| Teacher picks "harder" with grade=12 | Difficulty directive uses "include some grade-12 concepts" (clamped — no grade-13). |
| Teacher picks "easier" with grade=K (= "K" string) | Difficulty directive uses "below the current grade level" (non-numeric grade fallback). |
| Teacher picks "easier" with grade=1 | Clamped to grade-1 (max(1, 1-1) = 1; effectively no change in target grade — directive still says "simpler vocabulary"). |
| Body has `count=true` | 400 (booleans rejected). |
| Body has `count=2.5` | 400 (not integer). |
| Body has `count=20` | 400 (out of range). |
| Body missing `count` | Defaults to 8. |
| Body has `difficulty="hard"` (typo) | 400 ("must be one of: easier, same, harder"). |
| Body missing `difficulty` | Defaults to "same". |
| Personalized mode triggered, teacher picks count=10 | All N variants generated with count=10. Mix is uniform across variants. |
| Existing API caller (legacy frontend, external script) | Send no count/difficulty → server defaults → existing behavior. Backwards-compat preserved. |
| Drawer abandons mid-config (closes drawer before Generate) | No state to clean; no AI cost. |
| Adjust-settings during preview → config → cancel back | data/variants preserved (state machine restores preview without re-fetching). |
| Adjust-settings during preview → config → Generate | regenerate path with new params; replaces data/variants. |
