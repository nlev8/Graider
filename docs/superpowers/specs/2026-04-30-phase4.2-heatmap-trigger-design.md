# Phase 4.2 #10 — AssessmentComparison Heatmap Remediate Trigger

**Status:** Green-tier item from `project_phase4.2_backlog.md` #10. **Final** Phase 4.2 item.
**Differentiator vs Progress Learning:** PL has no contextual remediate-trigger from a comparison/heatmap view. Graider's AssessmentComparison heatmap now becomes a third RemediationDrawer trigger surface (joining Progress Rank cell-click and column-header hover-reveal).

## Goal

When the teacher views the AssessmentComparison heatmap and sees a red cell (class mean < 70% on a (content × standard) pair), clicking the cell opens the existing RemediationDrawer in `red_tier_in_class` mode for that standard. Resolves to the same class-wide red-tier flow as the Progress Rank column-header trigger.

## Locked decisions (from brainstorming + Codex mid-brainstorming reconciliation, 2026-04-30)

- **Q1:** Red cells only (`cell.percentage < 70`). Yellow + green cells are not clickable. Matches Progress Rank red-tier semantics.
- **Q2:** Whole cell is the click target. Mirrors ProgressRankGrid cell-click pattern.
- **Q3:** Always `red_tier_in_class` mode. Heatmap is content-level, no single-student scope.
- **Q4:** Pass `standard_code` only; drop content_id context. Remediation is standard-driven.
- **Q5:** `cursor: pointer` + native `title` tooltip + subtle hover outline on red cells (Codex MINOR — cursor + tooltip alone is weak discoverability). No icon overlay.
- **Q6:** Existing `/remediate` 422 handles "no red-tier students on this standard" via the drawer's error state. No new backend handling.
- **Q7:** AssessmentComparison.jsx mirrors ProgressRankGrid.jsx pattern: import RemediationDrawer + manage `remediationTrigger` state + mount drawer at the bottom + bump `reloadKey` on `onPublished`.
- **Q8:** Personalization (Phase 4.2 #2) and pre-gen config (Phase 4.2 #3) inherit from the existing drawer flow unchanged.
- **Q9:** No backend tests (no backend change). Manual smoke only.
- **Codex MAJOR reconciliation — UX copy:** The heatmap is content-specific, but `/remediate red_tier_in_class` resolves red-tier students across ALL class content (latest attempt). A student who aced other quizzes on the standard but bombed this specific cell's content may NOT be in the class-wide red tier — they wouldn't get a remediation. Tooltip + drawer subtitle must frame this clearly as "standard-wide class remediation" not "students who failed this cell." Existing drawer subtitle "for N red-tier students" is already semantically correct; the cell-level tooltip just needs to match.
- **Codex MINOR — Defensive bail:** Click handler explicitly checks `if (!isRed) return` before opening drawer.

## Architecture

**File:** `frontend/src/tabs/AssessmentComparison.jsx`. Frontend-only. No backend changes.

### State additions (mirroring ProgressRankGrid)

```js
import RemediationDrawer from "./RemediationDrawer";

// inside the component:
var [remediationTrigger, setRemediationTrigger] = useState(null);
var [reloadKey, setReloadKey] = useState(0);
```

### Cell rendering

The heatmap currently renders `<td>` cells with inline `style` and a `title` attribute. The new logic:

```js
var isRed = cell && typeof cell.percentage === "number" && cell.percentage < 70;
var cellTitle = isRed
  ? "Class mean " + cell.percentage + "% on " + code +
    ". Click to generate a class-wide remediation for this standard."
  : (existing tooltip per the cell state)

return (
  <td
    style={Object.assign({}, existingStyle, isRed ? {
      cursor: "pointer",
      // Hover outline added via CSS class, not inline (CSS :hover doesn't work inline).
    } : {})}
    className={isRed ? "phase4-heatmap-red-clickable" : ""}
    title={cellTitle}
    onClick={isRed ? function() {
      setRemediationTrigger({ standardCode: code });
    } : undefined}
    role={isRed ? "button" : undefined}
    tabIndex={isRed ? 0 : undefined}
    onKeyDown={isRed ? function(e) {
      if (e.key === "Enter" || e.key === " ") {
        e.preventDefault();
        setRemediationTrigger({ standardCode: code });
      }
    } : undefined}
  >
    {/* existing cell content */}
  </td>
);
```

### Hover styling (CSS-in-style block)

Add a `<style>{...}</style>` block alongside the existing heatmap render with:

```css
.phase4-heatmap-red-clickable:hover {
  outline: 2px solid var(--accent-primary);
  outline-offset: -2px;
}
.phase4-heatmap-red-clickable:focus {
  outline: 2px solid var(--accent-primary);
  outline-offset: -2px;
}
```

The existing heatmap uses inline styles only; one `<style>` block is the simplest way to add `:hover` (matches ProgressRankGrid's pattern of inlining a CSS string for hover-reveal at line ~338).

### Drawer mount

At the bottom of the component (before the existing return-close):

```jsx
{remediationTrigger && (
  <RemediationDrawer
    open={!!remediationTrigger}
    onClose={function() { setRemediationTrigger(null); }}
    classId={classId}
    standardCode={remediationTrigger.standardCode}
    targetMode="red_tier_in_class"
    targetStudentId={null}
    targetStudentName=""
    onPublished={function() { setReloadKey(function(k) { return k + 1; }); }}
  />
)}
```

### Reload integration

Add `reloadKey` to the existing fetch effect's dependency array so a successful publish triggers a re-fetch (matches ProgressRankGrid pattern at line ~334).

## Edge cases

| Case | Behavior |
|---|---|
| Cell is null/undefined ("not covered") | Not clickable; no `cursor: pointer`; no tooltip-trigger. The existing "Not covered" tooltip remains. |
| Cell percentage is null | Treated as not-red. Not clickable. |
| Cell percentage is 0 | `< 70` → red → clickable. (0% mastery is the most extreme red; clicking is sensible.) |
| Cell percentage is exactly 70 | Not red (`< 70` is strict). Not clickable. Matches Progress Rank threshold. |
| Cell is yellow (70-84) or green (≥85) | Not clickable. Existing tooltips remain. |
| Cell would resolve to zero red-tier students at the route level | `/remediate` returns 422 ("No red-tier students on this standard"). Drawer's error state surfaces it. Acceptable — rare edge case (heatmap mean < 70 means at least some students dragged the mean down; the class-wide red-tier resolution might still find them, just possibly not the ones whose performance on THIS cell drove the red mean). |
| Cell-level mean < 70 BUT class-wide latest mean on the same standard ≥ 70 | Confusing edge case. The drawer subtitle says "for N red-tier students" — the N is the class-wide red-tier count, NOT the cell's contributing students. Tooltip mitigates this by saying "class-wide remediation for this standard" (not "for the students who failed this cell"). Codex MAJOR reconciliation. |
| Teacher clicks during fetch | Drawer opens normally; existing drawer state machine handles in-flight fetches. |
| Heatmap re-renders with new data while drawer is open | Drawer's parent (AssessmentComparison) keeps `remediationTrigger` state independent of heatmap data. Drawer state preserved. |
| Onpublished re-fetch | Bumps `reloadKey`; existing fetch effect re-runs; heatmap data refreshes (new published_content row may slightly affect class means after students take it, though typically not immediately). |
| Keyboard accessibility | `role="button"` + `tabIndex={0}` + Enter/Space onKeyDown opens drawer. Same as Progress Rank pattern. |
| Heatmap renders 0 standards (all green) | Nothing to click. Tab stays informational. |

## Testing

**Backend:** No tests (no backend change).

**Frontend:** Manual smoke (Phase 4.2 precedent — no Vitest/RTL):
1. Open AssessmentComparison sub-tab → see heatmap.
2. Identify a red cell (cell mean < 70%) → cursor changes to pointer on hover; tooltip shows "Class mean N% on STANDARD. Click to generate a class-wide remediation for this standard."; subtle outline appears on hover.
3. Click red cell → drawer opens to "config" state (Phase 4.2 #3).
4. Confirm drawer subtitle: "for N red-tier students" (Phase 4 + Phase 4.2 #2 logic).
5. Pick count + difficulty → Generate → preview → Publish.
6. Verify `onPublished` triggers heatmap re-fetch (slight delay; heatmap data unchanged for this remediation publish, but the existing `getClassRemediationEffectiveness` data will pick it up later).
7. Click yellow/green cell → nothing happens (not clickable).
8. Click "not covered" cell (no submissions) → nothing happens.
9. Keyboard: Tab to red cell, press Enter → drawer opens.
10. Edge: red cell where the standard happens to have NO class-wide red-tier students → drawer opens to "config", teacher generates, /remediate returns 422 → drawer shows error state. Acceptable rare edge.

- Yellow-cell trigger surface. Q1 → red only.
- Icon overlay on red cells. Q5 → tooltip + cursor + hover outline only.
- "Remediate by content" alternative trigger that filters red-tier to students who specifically failed THIS cell. Out of scope; new feature.
