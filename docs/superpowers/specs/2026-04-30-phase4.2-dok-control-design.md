# Phase 4.2 #12 — DOK Control on Remediation Prompt

**Status:** Open follow-up surfaced during Phase 4.2 #2 brainstorming, filed and shipped as #12.
**Differentiator vs Progress Learning:** PL has DOK-tiered question banks but no per-remediation rigor selection. Graider's `/remediate` already has count + difficulty (Phase 4.2 #3); this slice adds DOK level (1-4 + Auto) so teachers can target cognitive demand explicitly.

## Goal

The pre-generation config dialog gets a NEW third row: a 5-button DOK toggle (`Auto / 1 / 2 / 3 / 4`). When teacher picks DOK 1-4, the AI prompt appends "Each question MUST be at DOK level N: {one-line description}." plus a clarifier that DOK and difficulty are orthogonal (DOK = cognitive rigor; difficulty = vocab/scaffolding tone; they can coexist). When teacher picks Auto (default), no DOK directive is added (preserves current behavior). AI returns per-question `dok: N` field that pass-through to `published_content.content` JSONB.

## Locked decisions (from brainstorming + Codex mid-brainstorming reconciliation, 2026-04-30)

- **Q1:** Single-target dropdown values, NOT distribution. Five options: Auto, 1, 2, 3, 4. Mixed-DOK is more assessment-style; remediation targets ONE skill at ONE rigor.
- **Q2:** New 3rd row in pre-gen config dialog under count + difficulty.
- **Q3:** Default = "Auto" (no DOK directive added). Backwards-compat: legacy callers without `dok` get current behavior.
- **Q4:** 5-button toggle matching the difficulty-button pattern. Use `flexWrap: "wrap"` for narrow viewports (Codex MINOR).
- **Q5:** One-line per-level DOK descriptions (NOT the verbose Planner block). Codex MINOR — condensed for the smaller drawer surface.
- **Q6 (Codex MAJOR):** DOK + difficulty coexist BUT prompt explicitly orders the two. Add a clarifier line: "Cognitive rigor is set by DOK; vocabulary and scaffolding tone are set by difficulty. They can coexist — e.g., DOK 4 + easier means extended thinking with simpler reading load." Prevents AI confusion on apparent contradictions.
- **Q7:** Prompt directive when DOK ∈ 1-4: "Each question MUST be at DOK level {N}: {ONE_LINE_DESCRIPTION[N]}." When Auto: no addition.
- **Q8:** AI returns `dok: N` per question; pass-through to JSONB. `_post_process_assignment` doesn't strip unknown fields. **Codex MINOR:** extend `_auto_fix_flagged_questions` (in `assignment_post_processing.py`) prompt to say "Keep the same DOK level and include the existing dok field" so auto-fix doesn't drift DOK when correcting other issues.
- **Q9:** Validation strictness mirrors count exactly: 400 on explicit invalid (out-of-range, non-int, bool); defaults to "Auto" only when missing.
- **Q10:** DOK uniform across all N variants in personalized mode. Same as count + difficulty pattern.
- **Q11:** Standard-metadata auto-pick out of scope.
- **Q12:** No dashboard / report changes. DOK is generation-time only.
- **Module constants (Codex MINOR):** `DOK_OPTIONS = (1, 2, 3, 4)`, `REMEDIATION_DOK_DEFAULT = None`, `DOK_DESCRIPTIONS = {1: "...", 2: "...", 3: "...", 4: "..."}` — single source for prompt + frontend.

## Architecture

### Backend

**File:** `backend/routes/student_portal_routes.py`.

#### New constants (alongside count/difficulty constants from #3)

```python
DOK_OPTIONS = (1, 2, 3, 4)
REMEDIATION_DOK_DEFAULT = None  # None = "Auto" (no DOK directive)

# Condensed one-line descriptions per Webb's DOK level. Used for both
# prompt directive and frontend tooltip (single source of truth).
# Codex MINOR: condensed from Planner's verbose block at planner_routes.py:3870-3897.
DOK_DESCRIPTIONS = {
    1: "Recall & Reproduction — facts, terms, simple procedures.",
    2: "Skills & Concepts — compare, organize, explain relationships, apply concepts.",
    3: "Strategic Thinking — analyze with evidence, justify reasoning, multi-step decisions.",
    4: "Extended Thinking — synthesize across sources, design solutions, sustained investigation.",
}
```

#### Update `_build_remediation_prompt`

Signature: `_build_remediation_prompt(*, grade, subject, standard_code, count=None, difficulty=None, dok=None)`.

Body extension when `dok in DOK_OPTIONS`:

```python
dok_directive = ""
if dok in DOK_OPTIONS:
    dok_directive = (
        f"Each question MUST be at DOK level {dok}: "
        f"{DOK_DESCRIPTIONS[dok]} "
        f"Cognitive rigor is set by DOK; vocabulary and scaffolding tone are "
        f"set by difficulty. They can coexist — e.g., DOK 4 + easier means "
        f"extended thinking with simpler reading load. Each question MUST "
        f"include a 'dok' integer field set to {dok}."
    )
```

Prepend `dok_directive` to the `QUESTIONS` block in the prompt. The order: DOK first (cognitive rigor), then difficulty (vocab tone), so the AI processes the rigor constraint before the tone modifier.

#### `/remediate` route validation (alongside count/difficulty validation block)

```python
# Phase 4.2 #12: validate optional dok param.
raw_dok = body.get('dok')
if raw_dok is None:
    dok = REMEDIATION_DOK_DEFAULT  # None = Auto
else:
    if isinstance(raw_dok, bool) or not isinstance(raw_dok, int):
        return error_response(
            f"dok must be an integer in {list(DOK_OPTIONS)}", 400,
        )
    if raw_dok not in DOK_OPTIONS:
        return error_response(
            f"dok must be one of: {', '.join(str(d) for d in DOK_OPTIONS)}", 400,
        )
    dok = raw_dok
```

Pass `dok` into both shared and personalized `_build_remediation_prompt` calls. Pass `dok` into `_gen_variant_for_student` worker (worker doesn't need it directly — already baked into base_prompt — but propagate for explicit traceability).

Actually — `dok` is already in `base_prompt` via the helper. Worker doesn't need a separate kwarg. Same pattern as `difficulty` in #3.

#### `_auto_fix_flagged_questions` prompt extension

**File:** `backend/services/assignment_post_processing.py` around line 1916-1941. Append to the auto-fix prompt's `Rules:` block:

```
- Keep the same DOK level. If the original question has a 'dok' field,
  the corrected question MUST include the same 'dok' field with the same
  value. Do NOT change DOK levels during auto-fix.
```

This prevents DOK drift when the auto-fix path mutates question text/answer/options for other quality issues.

#### `postRemediate` payload extension

Both `handleGenerate` and `regenerateAll` add `dok: configDok` to the payload. When configDok is null, the field is sent as null; backend treats null as missing → Auto behavior.

### Edge cases

| Case | Behavior |
|---|---|
| Body missing `dok` | Defaults to "Auto"; no DOK directive in prompt. Backwards-compat. |
| Body has `dok=null` explicitly | Same as missing — defaults to Auto. |
| Body has `dok=true` | 400 (booleans rejected, mirrors count). |
| Body has `dok=2.5` | 400 (not integer). |
| Body has `dok=5` | 400 (out of range). |
| Body has `dok="2"` | 400 (string, not int). |
| Auto in drawer + Generate | `dok: null` in payload; no DOK directive in prompt; AI free to pick. |
| DOK 4 + difficulty=easier | Prompt clarifies they coexist ("DOK 4 + easier means extended thinking with simpler reading load"). AI treats rigor as primary, tone as secondary. |
| AI returns question without `dok` field | Field absent from JSONB. Acceptable — Auto mode permits this; for explicit DOK requests, the prompt directive is best-effort. No 422 floor. |
| AI returns question with `dok=5` (out of valid range) | Pass-through to JSONB. UI doesn't display DOK; no validation done on output. Acceptable — generation is the AI's job. |
| Personalized mode | DOK uniform across all N variants. Each variant's prompt includes the same DOK directive. |
| Auto-fix path corrects a question | New `_auto_fix_flagged_questions` prompt rule preserves DOK field on corrected output. |
| `_post_process_assignment` runs | Doesn't strip `dok` field (it doesn't whitelist question fields). Pass-through preserved. |
| Effectiveness dashboard rendering | DOK not displayed; no changes needed. |
| Recall / weekly cap / gradebook | DOK doesn't affect any of these. Inherits unchanged. |

## Out of scope (deferred)

- Per-DOK distribution (Planner-style `{1: 2, 2: 4, ...}`). Single-target is sufficient for remediation.
- Standard-metadata auto-pick. Teacher picks manually for now.
- DOK display on the effectiveness dashboard / gradebook / submission detail. Out of scope; existing surfaces don't display DOK.
- "DOK aware" cap or rate-limit. Phase 4.2 #8 caps don't change.
- Replacing the difficulty toggle with DOK alone. Difficulty + DOK are orthogonal (vocab/scaffolding vs cognitive rigor); both stay.
- Per-question DOK editing in the drawer preview. Out of scope; teachers can regenerate if needed.
- Configurable `DOK_OPTIONS` env var. Hardcoded 1-4 (Webb's DOK is fixed).
- `_auto_fix_flagged_questions` testing — the prompt extension is best-effort; no automated assertion. Existing `_auto_fix_flagged_questions` already gracefully handles auto-fix failures via the broad try/except.
