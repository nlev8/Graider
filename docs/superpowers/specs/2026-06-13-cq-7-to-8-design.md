# Code Quality 7 → 8 Campaign — Design

**Date:** 2026-06-13
**Author:** brainstorm session (controller + operator)
**Status:** Approved design → next step `writing-plans`
**Classification:** mixed — bulk is **Class A** (behavior-preserving extraction); the
grading-critical subset is **Class B** (manual merge, golden-grading regression gate).

---

## 1. Goal & success criterion

Flip the **Code Quality** dimension of the hardening rubric from **7 → 8**, honestly
and provably, with **zero behavior change**.

The level-8 anchor (`docs/superpowers/specs/2026-06-02-hardening-rubric-anchors.md` §3):

> **Level 8 = No file >2,500 LOC; no function >200 LOC** (applies to ALL source files,
> FE + BE).

**Hard success criterion (strict standard — chosen in brainstorm):** a re-run of both
function scans plus the anchors' literal `find … | xargs wc -l` command shows
**0 files >2,500 LOC** and **0 functions >200 LOC** across `backend/` + `frontend/src/`,
**including test files and SVG math components — no exceptions.** Level 8 has no
"documented exception" clause (unlike Architecture level 9), so any residual offender
fails the flip.

**Score effect:** CQ 7 → 8; overall mean ≈ +0.1 (**7.85 → ~7.95**). This is a *threshold,
not a gradient* — splitting 126 of 127 functions banks exactly zero. The campaign is
all-or-nothing for the point, but additive/reversible at every step (every merged PR is
pure structure improvement even before the flip).

## 3. Scan tooling (reproducible)

**Backend (Python AST):** walk `backend/`, `ast.parse` each `*.py`, flag any
`FunctionDef`/`AsyncFunctionDef` whose `end_lineno - lineno + 1 > 200`.

**Frontend (Babel):** walk `frontend/src/`, `@babel/parser` (sourceType module, jsx
plugin) each `*.{js,jsx}`, flag any `FunctionDeclaration`/`FunctionExpression`/
`ArrowFunctionExpression` whose `loc.end.line - loc.start.line + 1 > 200`.

**Files:** `find backend frontend/src \( -name '*.py' -o -name '*.jsx' -o -name '*.js' \)
| xargs wc -l | awk '$1>2500'`.

Both scan scripts are committed to `scripts/` in Wave 0 so the re-score (and future
audits) are reproducible, not ad-hoc.

## 6. Verification nets

**Shared baseline (every PR):**
- Full `pytest -q --ignore=tests/load` green (baseline ~5966 passed, 16 skipped).
- `grep -rln '<each modified file>' tests/` → run every surfaced test.
- **SIS pin scan** — `tests/test_sis_alerting.py` pins exact line numbers in
  `clever_routes.py` (332/389/883/925); Wave 7 splits `clever_callback`, so pin updates
  with documenting comments are mandatory there. Any other pin tests surfaced → same.
- FE PRs: `npx vitest run` + `npm run build`. BE PRs: `ruff check` + `bandit -q -r` on
  changed files.

**Tier A proof — behavior-preserving extraction:**
- **Pure-move discipline:** extracted helper takes exactly the inputs the inline code
  used, returns exactly what it produced; no logic change, no side-effect reordering
  across the boundary. The "moving, not adding" tell must answer *moving*.
- **opus reviewer** confirms pure-move; **Codex adversarial pass**
  (`codex exec -s read-only -c model_reasoning_effort=high`) checks the Hard Rule #11
  hazards: identifier capture, eager evaluation at guard seams, stale closures,
  non-pure-move logic.
- Existing golden + the component's `.mount.test.jsx` in the touched path stay green. If
  a split FE component has no mount test, adding one is part of that PR.

**Tier B proof — Tier A PLUS (existing nets, reused as hard gates):**
- **Golden grading:** `tests/test_grader_golden.py`, `test_grading_thread_golden.py`,
  `test_portal_grading_golden.py` — identical scores & feedback before/after.
- **Prompt-snapshot:** `test_grader_prompt_snapshots.py`, `test_grading_prompt_snapshot.py`,
  `test_grading_pipeline_helpers.py` — assembled prompt byte-identical (the mechanical
  "no factor dropped from the prompt" check).
- **Deterministic LLM:** mocked via existing `tests/grading_fakes.py`.
- **Factor-coverage checklist in the PR body:** for each grading-fn split, enumerate
  which of the 18 factors flow through it and confirm each still flows post-split.
- **Build-if-missing (Wave 8, task 1):** confirm every Tier B function is actually
  *covered* by a golden/snapshot test. If any isn't, add a pinning test (proven to
  capture current behavior) **before** splitting it. Never split a grading function that
  nothing pins.

## 7. Per-PR workflow & merge gating

1. Implementer (worktree subagent, inherits main-loop model) does the pure-move split,
   inner-loop tests red→green.
2. **Spec-compliance review first** (opus): only the target functions split, nothing
   extra, no behavior change attempted.
3. **Code-quality review** (opus): pure-move verified, no hazards.
4. **Codex adversarial pass** — standing second-model layer (Hard Rule #11).
5. Fix loop until both clear; controller (main loop) arbitrates — never delegated.
6. **Layer 3 gates:** full pytest, cross-cutting grep, pin scan.
7. Merge: Tier A auto-merge on green; Tier B manual controller merge after all gates +
   golden regression clean.

**Commit hygiene:** each PR carries only its split's files — never `flask_session/`,
`.claude/scheduled_tasks.lock`, handoff/AGENTS.md/gitnexus noise.

## 8. Failure / rollback

- A split a net flags as behavior-changing is **rejected pre-merge** — blast radius one
  file.
- A merged Tier A PR that later regresses → revert its single squash commit (each PR is
  one self-contained file split → clean revert).
- Tier B golden regression failing *means* scores changed → does not merge, full stop.
- The campaign is additive/reversible: stop at any wave and everything merged is pure
  structure improvement (the score only flips at 100%).

## 9. Definition of done

1. Both scans + the anchors' literal verify command → **0 files >2,500, 0 functions
   >200** (strict; no excluded files). Raw output captured.
2. Full `pytest -q --ignore=tests/load` green on `main`; `npx vitest run` + `npm run
   build` green.
3. **GitNexus reindex** (`npx gitnexus analyze --embeddings --skip-agents-md`).
4. **Re-score commit** (docs-only, Class A): anchors CQ row 7→8 with re-scan evidence
   inline; overall 7.85 → ~7.95; per-dimension spot-check that nothing else moved. Gated
   by opus adversarial + Codex pass.
5. **Evidence ledger** in the re-score PR body: before (1 file + 127 fns) → after (0 + 0),
   test counts, full wave/PR index.

## 10. Non-goals

- No level-9 work — the 7 files in the 2,000–2,500 LOC band stay (Appendix B).
- No non-CQ dimension.
- Zero behavior change. (If a split *requires* a behavior change to fit ≤200, that's a
  red flag — stop and re-plan, don't smuggle logic into a "refactor.")
